# Create Workflow Prompt

Create a new workflow under `workflows/[workflow-name].md`.

The workflow must include:
- purpose
- trigger
- inputs
- primary agent
- supporting agents
- skills used
- vault context used
- procedure
- output (path under `outputs/`)
- approval points

A workflow combines agents, skills, and vault context into a repeatable business process.

Rules:
- Reference existing agents and skills by name.
- Do not invent skills or agents that do not exist.
- Define a clear output location.
- Mark approval points explicitly.
