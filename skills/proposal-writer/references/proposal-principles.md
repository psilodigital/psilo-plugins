# Proposal Principles

How Psilodigital writes proposals.

## Voice

- Direct, builder-led, no corporate filler.
- Match the client's vocabulary, not ours.
- Read it aloud — if it sounds like marketing, rewrite it.

## Structure

- One screen of summary up top. The client should be able to decide if they want to read on.
- Problem first. Solution second. Price last.
- One CTA at the end.

## Scope discipline

- Every proposal includes an explicit "Out of scope" section.
- Phases must end in something an operator can verify.
- Always include assumptions. If we are guessing, say so.

## Pricing posture

- Fixed-price when scope is clear.
- Ranges when scope is not yet final.
- Never quote a final number without operator sign-off.
- Separate one-time build cost from monthly run cost.

## What we avoid

- Fabricated case studies.
- Vague claims ("transformational", "AI-powered").
- Pricing that hides assumptions.
- Promising specific savings without data.
- Copying old proposals without re-checking applicability.
