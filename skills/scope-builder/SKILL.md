# Scope Builder

## When to use

Use this skill when turning requirements into:
- MVP scope
- feature list
- technical scope
- delivery phases
- implementation assumptions
- risks and dependencies

## Inputs

- Business problem
- Client context
- Desired outcome
- Existing tools
- Known integrations
- Timeline constraints
- Budget constraints

## Procedure

1. Identify the business goal.
2. Translate goals into system capabilities.
3. Separate MVP from later phases.
4. Identify integrations.
5. Identify data entities.
6. Identify roles and permissions.
7. Identify risks and dependencies.
8. Produce a delivery-ready scope.

## Output format

Return:

1. Business goal
2. MVP scope
3. Out-of-scope items
4. User roles
5. Core features
6. Data model overview
7. Integrations
8. Risks
9. Assumptions
10. Suggested phases

## Rules

- Do not assume API access exists.
- Flag unknown integration feasibility.
- Prefer simple MVP scope.
- Avoid overengineering.
