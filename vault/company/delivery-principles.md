# Delivery Principles

How Psilodigital delivers work.

## Default rhythm

- Short phases. Each phase ships something usable.
- Weekly client check-in.
- Demo at the end of each phase.
- Written decision log.

## Build choices

- Start with the smallest slice that proves the outcome.
- Prefer boring tech that the client can live with for years.
- Avoid premature complexity.
- Document every external integration.

## When things change

- Scope changes go through a written change request.
- Slippage is communicated early. Never silent.
- Risks are surfaced as they appear, not at the end.

## When we hand over

- Documentation written for the operator, not for engineers.
- Credentials owned by the client.
- A clear path for support after delivery.
