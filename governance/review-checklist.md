# Review Checklist

Use this before merging changes.

## Agent checklist

- Clear role
- Clear responsibilities
- Allowed skills listed
- Allowed vault folders listed
- Restricted actions listed
- Approval needs defined

## Skill checklist

- Has SKILL.md
- Has name and description metadata
- Has activation conditions
- Has inputs
- Has procedure
- Has output format
- Has examples or templates
- Does not include secrets

## Client context checklist

- Facts separated from assumptions
- Source/date included when possible
- No unrelated client data
- Sensitive data marked clearly

## Workflow checklist

- Trigger is clear
- Inputs are clear
- Agent is clear
- Skills are clear
- Output location is clear
- Approval points are clear
